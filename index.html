O projeto GCI

Introdução

Este projeto foi desenvolvido como parte de nossa disciplina e teve como foco a instalação e configuração de um servidor WEB. O objetivo principal foi criar um servidor capaz de hospedar uma página acessível remotamente, abordando o tema "Máscara de IP". Durante o processo, aprendemos a instalar e configurar o serviço, desenvolver a página e garantir sua segurança e funcionalidade.

Um servidor WEB é um software responsável por receber solicitações de navegação e responder enviando páginas HTML, imagens ou outros conteúdos para o cliente (navegador). Ele é essencial para qualquer aplicação que dependa da internet, como sites e sistemas online.

Softwares Nescessários

Para este trabalho, utilizamos uma extensão do Visual Studio Code (VSCode), chamada Five Server, como ferramenta para configurar o servidor WEB. Optamos por ele devido à sua simplicidade e integração direta com o ambiente de desenvolvimento utilizado, o que facilitou o desenvolvimento do projeto. Algumas características incluem:

  • Recarga automática (Live Reload): Atualizações em tempo real ao salvar mudanças no código.
  • Compatibilidade com HTML, CSS e JavaScript: Ideal para desenvolvimento de páginas web simples e dinâmicas.
  • Configuração simples: Não requer instalação complexa ou ajustes avançados.

Além disso, estudamos outro software alternativo, o Apache HTTP Server, que possui características avançadas, como suporte a protocolos HTTP e HTTPS, além de sua flexibilidade para hospedar desde páginas simples até aplicações complexas. Embora não o tenhamos implementado neste projeto, ele foi avaliado como uma possível solução para ambientes com maior volume de tráfego ou necessidades específicas.

Configuração do servidor

Para a configuração do nosso servidor, o primeiro passo foi instalar o Five Server no VSCode. É válido citar que o sistema operacional utilizado para a realização do projeto foi o Ubuntu 22.04 LTS. Após a instalação, configuramos a porta padrão, sendo ela a porta 5500. Criamos também um arquivo raiz, chamado index.html para hospedar a página.

Desenvolvimento da página WEB

Criamos uma página em HTML, CSS e JavaScript para demonstrar o funcionamento do servidor WEB. Ela foi dividida nas sessões visíveis ao lado, sendo elas Introdução, Softwares Utilizados, Desenvolvimento e Conclusão. A página foi testada em dispositivos conectados à mesma rede e acessada pelo IP do servidor.

Testes Realizados

Realizamos testes para verificar a acessibilidade e funcionalidade do servidor:

  • Ping:Para garantir que o servidor estava alcançável na rede.
  • Acesso pelo navegador:Testamos a página a partir de dispositivos diferentes.
  • Firewall:Verificamos bloqueios e ajustamos regras para liberar o tráfego necessário.

Incluímos também o uso de tracert para identificar o caminho dos pacotes até o servidor e confirmar a conectividade.

Concluindo

A realização deste trabalho foi uma experiência enriquecedora que nos permitiu aplicar conceitos teóricos em um ambiente prático. A configuração do servidor WEB nos mostrou a importância de cada detalhe técnico, desde a instalação até a segurança.

O uso do versionamento com Git foi um diferencial, permitindo que todos os membros colaborassem de forma organizada e eficiente. Este aprendizado será valioso em projetos futuros, especialmente em ambientes colaborativos.

Estamos satisfeitos com os resultados, mas reconhecemos que há áreas a serem aprimoradas, como a implementação de HTTPS e o uso de scripts para automatizar tarefas. Este projeto nos preparou melhor para lidar com desafios reais no mundo da tecnologia e reforçou a importância de boas práticas em redes e servidores.
